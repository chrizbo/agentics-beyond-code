## How to Read This

Each weekly entry shows which strategic priorities had calendar coverage, which
had none, and whether async GitHub activity filled any gaps. A zero-coverage
priority is a conversation starter, not an accusation — async work often
advances strategy without requiring meetings.

**Coverage key:** 🟢 > 30 min | 🟡 1–30 min | 🔴 0 min (meeting or async)
